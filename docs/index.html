EnvChecker

A powerful environment variable validator for Node.js applications

Get Started View on GitHub

Features

Type Validation

Validate strings, numbers, URLs, and booleans with built-in type checking.

Pattern Matching

Use regular expressions to validate variable formats and patterns.

Number Constraints

Set minimum and maximum values for numeric environment variables.

Default Values

Specify default values for optional environment variables.

Error Handling

Clear and descriptive error messages for validation failures.

TypeScript Support

Full TypeScript type definitions for better development experience.

Installation

Install using npm:

npm install @envchecker/env-validator

Usage

Basic configuration example:

const { validateEnv } = require('@envchecker/env-validator');

const config = {
    REQUIRED_VARIABLES: {
        PORT: {
            type: 'number',
            required: true,
            min: 1024,
            max: 65535
        },
        API_KEY: {
            type: 'string',
            required: true,
            pattern: '^[A-Za-z0-9]{32}$'
        },
        DEBUG: {
            type: 'boolean',
            required: false,
            default: false
        }
    }
};

try {
    const { validatedEnv } = validateEnv(config);
    console.log('Environment variables validated successfully:', validatedEnv);
} catch (error) {
    console.error('Validation failed:', error.errors);
    process.exit(1);
}

API Reference

Configuration Options

type

Type of the environment variable. Supported values: 'string', 'number', 'boolean', 'url'

required

Whether the variable is required. Defaults to true.

default

Default value if the variable is not set.

pattern

Regular expression pattern for string validation.

min/max

Minimum and maximum values for number validation.
